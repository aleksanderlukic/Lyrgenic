# Lyrgenic

AI-powered lyrics generator for beats and instrumentals. Upload your beat, choose a genre, vibe, language, and topic — and get fully structured, timestamped lyrics generated by GPT-4o.

---

## Tech Stack

| Layer     | Technology                                          |
| --------- | --------------------------------------------------- |
| Framework | Next.js 16 (App Router) + TypeScript                |
| Styling   | Tailwind CSS v4                                     |
| Auth      | Auth.js v5 (next-auth) — Credentials + Google OAuth |
| ORM       | Prisma + PostgreSQL                                 |
| Queue     | BullMQ + Redis (ioredis)                            |
| Storage   | AWS S3 / Cloudflare R2                              |
| LLM       | OpenAI GPT-4o                                       |
| Audio     | ffprobe (duration analysis)                         |
| Payments  | Stripe subscriptions                                |

---

## Prerequisites

- **Node.js** 18+
- **PostgreSQL** database
- **Redis** (optional — queues fall back to in-process if unavailable)
- **AWS S3** or **Cloudflare R2** bucket
- **OpenAI** API key
- **Stripe** account (for billing features)

## Setup

### 1. Clone & install

```bash
git clone <your-repo-url>
cd Lyrgenic
npm install
```

### 2. Environment variables

```bash
cp .env.example .env.local
```

Fill in every value in `.env.local`. Key variables:

| Variable                       | Description                               |
| ------------------------------ | ----------------------------------------- |
| `DATABASE_URL`                 | PostgreSQL connection string              |
| `NEXTAUTH_SECRET`              | Random secret (`openssl rand -base64 32`) |
| `NEXTAUTH_URL`                 | Base URL, e.g. `http://localhost:3000`    |
| `OPENAI_API_KEY`               | Your OpenAI API key                       |
| `REDIS_URL`                    | Redis URL (optional)                      |
| `AWS_ACCESS_KEY_ID` / `SECRET` | S3 credentials                            |
| `AWS_S3_BUCKET`                | Bucket name                               |
| `STRIPE_SECRET_KEY`            | Stripe secret key                         |
| `STRIPE_WEBHOOK_SECRET`        | Stripe webhook signing secret             |
| `GOOGLE_CLIENT_ID` / `SECRET`  | For Google OAuth (optional)               |

### 3. Set up the database

```bash
npm run db:generate   # generate Prisma client
npm run db:migrate    # apply migrations (creates tables)
```

### 4. Start development server

```bash
npm run dev
```

### 5. Start background worker (optional)

In a separate terminal:

```bash
npm run worker
```

> If Redis is not configured, jobs run in-process automatically — no separate worker process needed.

## Deployment (Vercel + Neon + Upstash)

1. Push repo to GitHub and import into Vercel
2. Add all `.env.local` values as Vercel environment variables
3. Provision a **Neon** PostgreSQL database; use the connection pooler URL as `DATABASE_URL`
4. Provision an **Upstash Redis** instance and set `REDIS_URL`
5. Run `npm run db:push` (or `db:migrate`) to create tables
6. Register a Stripe webhook pointing to `https://<your-domain>/api/stripe/webhook` with events:
   - `checkout.session.completed`
   - `customer.subscription.updated`
   - `customer.subscription.deleted`
